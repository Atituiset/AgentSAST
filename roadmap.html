路线图与已知局限

版本路线图

v0.1 (当前)

  • L1: Semgrep + Flawfinder + 内置模式扫描器
  • L2: Tree-sitter AST 解析 + 后向切片 + 结构体提取 + 调用图
  • L3: OpenAI 兼容 LLM 裁判 + 反幻觉提示
  • Pipeline 编排 + Click CLI
  • SARIF v2.1.0 统一输出
  • 12 个测试用例全部通过

v0.2 — 深度分析

  • CodeQL SARIF 集成 (L1 新增扫描器)
  • Joern CPG 深度切片 (L2 可选后端)
  • 多线程并行扫描
  • SARIF 结果合并 (跨工具统一输出)

v0.3 — 编译感知

  • Clang 预处理管线 (宏展开 + 精确类型)
  • Clang AST dump 解析
  • 编译数据库 (compile_commands.json) 支持
  • 头文件搜索路径配置

v0.4 — 工程化

  • 批量扫描 (多目标并行)
  • 增量分析 (仅分析 git diff 变更)
  • 基线管理 (已知误报标记)
  • 结果缓存 (避免重复 LLM 调用)
  • CI/CD 集成 (GitHub Actions, GitLab CI)

v0.5 — 平台化

  • Web 仪表盘 (漏洞管理 + 趋势分析)
  • GitHub Pull Request 评论集成
  • SARIF 上传到 GitHub Code Scanning
  • 多语言支持 (Rust, Go, Java)
  • 自定义规则 DSL

局限影响计划解决

技术债务

  1. _file_cache 无失效机制 — 长会话中文件修改后缓存不一致
  2. SlicingResult.to_dict() 不包含 callee_slices — 序列化不完整
  3. 跨文件搜索的 rglob("*") 性能 — 大项目需要文件索引
  4. 无配置文件 — 所有参数通过 CLI 传递，缺少 .agentsast.toml
  5. 无日志持久化 — 仅终端输出，无文件日志

贡献指南

欢迎贡献！优先级最高的改进方向：

  1. CodeQL SARIF 解析器 — 扩展 L1 覆盖率
  2. 配置文件支持 — .agentsast.toml 项目配置
  3. 增量分析 — git diff 模式，仅分析变更代码
  4. 结果缓存 — 避免重复 LLM 调用，降低成本
