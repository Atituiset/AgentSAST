## 已知局限

### Tree-sitter 相关

| 局限 | 影响 | 计划解决 |
|------|------|----------|
| 不展开 C 宏 | 宏内危险调用不可见 | v0.3 Clang 预处理 |
| 无类型推断 | `void*` 参数无法确定实际类型 | v0.3 Clang AST |
| 匿名结构体处理 | 部分边缘 case 可能漏提取 | 持续改进 |
| 条件编译 `#ifdef` | 所有分支都被解析 | v0.3 Clang 预处理 |

### 切片引擎相关

| 局限 | 影响 | 计划解决 |
|------|------|----------|
| 指针别名追踪 1-2 层 | 多级间接赋值可能遗漏 | v0.2 Joern CPG |
| 跨文件搜索全量扫描 | 大项目性能差 | v0.4 索引优化 |
| C++ 虚函数/函数指针 | 无法追踪动态分发 | v0.2 Joern CPG |
| 全局变量切片 | 当前仅追踪局部变量 | v0.2 改进 |

### LLM 裁判相关

| 局限 | 影响 | 计划解决 |
|------|------|----------|
| 幻觉未完全消除 | 可能误判 | 多轮验证 + 自一致性 |
| 无多轮对话 | 复杂漏洞需追问 | v0.3 对话模式 |
| Token 成本 | 大项目全量扫描昂贵 | v0.4 缓存 + 增量 |
| 置信度阈值固定 | 不同项目可能需要不同阈值 | v0.4 可配置阈值 |

### Flawfinder/内置扫描器相关

| 局限 | 影响 | 计划解决 |
|------|------|----------|
| 无数据流分析 | 高误报率 (设计权衡) | L2+L3 裁定 |
| 函数级匹配 | 无法检测逻辑漏洞 | v0.2 CodeQL |
| 23 个危险函数 | 覆盖不全 | 社区扩展 |

## 技术债务

1. `_file_cache` 无失效机制 — 长会话中文件修改后缓存不一致
2. `SlicingResult.to_dict()` 不包含 `callee_slices` — 序列化不完整
3. 跨文件搜索的 `rglob("*")` 性能 — 大项目需要文件索引
4. 无配置文件 — 所有参数通过 CLI 传递，缺少 `.agentsast.toml`
5. 无日志持久化 — 仅终端输出，无文件日志

## 贡献指南

欢迎贡献！优先级最高的改进方向：

1. **CodeQL SARIF 解析器** — 扩展 L1 覆盖率
2. **配置文件支持** — `.agentsast.toml` 项目配置
3. **增量分析** — `git diff` 模式，仅分析变更代码
4. **结果缓存** — 避免重复 LLM 调用，降低成本
